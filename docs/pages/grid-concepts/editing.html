Editing

FG-Grid provides several editors: string, number, boolean.
To enable an editor, a column should have the property editable: true.


columns: [{
  index: 'brand',
  type: 'string',
  editable: true
},{
To avoid repeating this property for all columns, there is a default option property. 

new Grid({
  defaults: {
    editable: true
  },
  columns: [{
    index: 'brand',
    type: 'string',
    editable: true
  },{

Editing for column without data index

There are 4 interconnected column properties:
format, getter, render, setter.
setter helps to store the value changed in the editor.

Sometimes it is required to get a value for a column in a non-standard way.
To read from another data index or to combine with several data indexes.
For this case, the combination of properties should be used:
format, getter, setter, render.
Let's study the simplest case when we do not set index.

getter is used for getting the data value for the editor field.

setter is used for setting the value for the editor field to the data store.

render is used to display our data in the cell.

Examples


columns: [{
  type: 'string',
  title: 'Model',
  render(params) {
    return params.item.model;
  },
  getter(params) {
    return params.item.model;
  },
  setter(params) {
    params.item.model = params.newValue;
  }
},{
