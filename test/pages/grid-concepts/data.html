Data

FG-Grid is oriented on using indexes(keys) of data.

JSON data

Example: array of objects 


data: [
  { brand: "Lexus", model: "RX 350", price: 60000, year: 2021 },
  { brand: "Lexus", model: "NX 300", price: 50000, year: 2023 },
  { brand: "Toyota", model: "Land Cruiser Prado", price: 70000, year: 2022 },
  { brand: "Toyota", model: "RAV4", price: 35000, year: 2023 },
  { brand: "Volkswagen", model: "Tiguan", price: 38000, year: 2021 }
]

Indexes are: brand, model, price, year.

In columns:


columns: [{
  index: 'brand',
  title: 'Brand',
  type: 'string'
},{
  index: 'model',
  title: 'Model',
  type: 'string'
},{
  index: 'price',
  title: 'Price',
  type: 'number'
},{
  index: 'year',
  title: 'Year',
  type: 'number'
}]

Column without data index

Sometimes it is required to get a value for a column in a non-standard way.
To read from another data index or to combine with several data indexes.
For this case, the combination of properties should be used:
format, getter, setter, render.
Let's study the simplest case when we do not set index.

getter is used for getting the data value for the editor field.

setter is used for setting the value for the editor field to the data store.

render is used to display our data in the cell.

Examples


columns: [{
  type: 'string',
  title: 'Model',
  render(params) {
    return params.item.model;
  },
  getter(params) {
    return params.item.model;
  },
  setter(params) {
    params.item.model = params.newValue;
  }
},{
